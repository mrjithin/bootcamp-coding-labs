Unique Pointer
--------------

Q1. List one or two benefits of using RAII to manage memory instead manually calling `new` and `delete`.
A1. By using RAII, we don't have to manually deallocate memory after use and it's done automatically
    when the object goes out of scope. The code also becomes less error-prone and there is less boilerplate
    to manage. 

Q2. When implementing move semantics for a `unique_ptr`, for example in the move constructor `unique_ptr(unique_ptr&& other)`, it is essential that we set the underlying pointer of the `other` parameter to `nullptr` before exiting the function. Explain in your own words what problem would arise if we did not.
A2. If we don't set it to nullptr, then there will be double deletion. When the moved object goes out of scope delete gets called once. 
    But when the newly created object goes out of scope, we delete the same memory location again resulting in double deletion. 

Q3. This method of recursive deallocation through RAII works great for small lists, but may pose a problem for longer lists. Why? Hint: what is the limit for how "deep" a recursive function's call stack can grow?
A3. When the list becomes huge, the recursive deallocation results in a huge stack depth resulting in stack overflow. Instead we should use iterative.

Q4. What does `std::move` do in this context? Why is it safe to use `std::move` and move semantics here?
A4. std::move moves the underlying pointer from the original unique_ptr to the new one while making the original point to nullptr.
This is necessary and safe since we don't want multiple unique_ptr to point to the same one as it will cause issues such as double free. 

Q5. Why does `operator->` return `T*` (a pointer) rather than `T&` (a reference)? Walk through what the compiler does when it sees `ptr->member`.
A5. The compiler recursively calls -> until it returns a pointer after which it calls the normal -> operator (equivalent to (*).). Hence we return a pointer
instead of a reference. 

Q6. Why does this class need BOTH `T& operator*()` and `const T& operator*() const`? Give a one-line code example that would fail to compile if one of them were missing, and explain which one.
A6. We need both so that we can make both const and non-const references to the unique_ptr. auto func(const unique_ptr<int>& a) { return *a;} will not run if the const version is absent.  

Q7. We marked `operator bool()` as `explicit`. What kind of unintended code would compile if we had left the `explicit` off? Give a concrete example.
A7. If we did not mark explicit, the unique_ptr will convert to a bool implicitly in all cases requiring an integer which is unintended. 
Example, unique_ptr<int> ptr(new int(3)); int b=30+ptr; will give b=33.
